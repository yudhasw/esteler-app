1 Overview

Project ini awalnya dibangun dengan Flask + SQL Server dan di-deploy via Docker. Untuk efisiensi biaya dan kemudahan deployment, sistem di-refactor menjadi serverless di Vercel dengan database Neon PostgreSQL (gratis 0.5 GB, scale-to-zero, auto-suspend).

🎯 Goals Refactor

  • Performa lebih cepat — query optimization, eager loading, indexing
  • Sesuai SDD final — tambah field payment_method, order_source, expires_at, status "siap"
  • Match desain Figma — UI mobile-first, hijau "Dapur Hijrah", bottom nav
  • Siap serverless — entry point Vercel, connection pooling, env config
  • Maintainable — service layer pattern, blueprint separation, DRY

2 Masalah di Code Lama

Analisis terhadap repo github.com/yudhasw/esteler-app menemukan 10 bug kritis yang harus diperbaiki sebelum deploy:

1

Model mismatch

Routes buyer.py & seller.py masih pakai model lama (Seller, is_available) padahal models.py udah baru. Bakal crash saat runtime.

2

User class tidak extend UserMixin

Flask-Login butuh UserMixin untuk method is_authenticated, get_id(). Login pasti error.

3

cart_service.get_summary() terpotong

Code berhenti di tengah: menu = Menu.query. Tidak ada return value, akan SyntaxError.

4

File services hilang

analytics_service.py dan recommendation_service.py diimport tapi tidak ada filenya. ImportError saat startup.

5

Template kosong (0 byte)

base.html, dashboard.html, login.html, menu.html semua file kosong. Halaman blank.

6

Blueprint tidak diregister

buyer_bp dan seller_bp didefine tapi tidak di-register di app.py. Route-nya tidak akan accessible.

7

N+1 query problem

Cart loop akses item.menu = 1 query per item. Untuk 10 item = 11 query. Lambat di serverless.

8

SECRET_KEY default insecure

Pakai 'dev-secret-key' sebagai fallback. Bahaya kalau production lupa set env var.

9

Field SDD belum ada

Model belum punya payment_method, order_source, expires_at, status "siap", walk-in order, dll.

10

Belum siap Vercel

Tidak ada vercel.json, pakai Gunicorn (untuk traditional server), connection pool tidak dikonfigurasi untuk serverless.

5 Setup Neon PostgreSQL

1 Daftar Neon

Buka neon.com → sign up pakai GitHub (gratis 0.5 GB).

2 Buat Project Baru

  • Project name: esteler-app
  • Region: AWS Singapore (ap-southeast-1) ← penting, deket Vercel sin1
  • PostgreSQL version: 16 (default)

3 Copy Connection String (POOLED!)

Di dashboard, klik "Connection Details". PENTING: aktifkan toggle "Pooled connection".

# Format pooled (untuk serverless Vercel)
postgresql://user:pass@ep-xxx-pooler.ap-southeast-1.aws.neon.tech/dbname?sslmode=require

# Format direct (untuk migrasi/seeding)
postgresql://user:pass@ep-xxx.ap-southeast-1.aws.neon.tech/dbname?sslmode=require

⚠️ Pooled string punya -pooler di hostname.

4 Jalankan Migration 

# Set env var dengan DIRECT connection (bukan pooled)
export DATABASE_URL="postgresql://...neon.tech/dbname?sslmode=require"

# Run migration
flask db upgrade

# Seed sample data (admin + 8 menu)
python seed.py

6 Setup Vercel

1 Push ke GitHub 

git add .
git commit -m "refactor: ready for vercel deploy"
git push origin main

2 Import Project di Vercel

  • Buka vercel.com → sign in dengan GitHub
  • "Add New Project" → pilih repo esteler-app
  • Framework Preset: Other (Vercel auto-detect via vercel.json)
  • Root directory: ./ (default)

3 Set Environment Variables

Di Settings → Environment Variables, tambahkan:

KeyValue
DATABASE_URLpostgresql://...pooler...neon.tech/... (POOLED!)
SECRET_KEYgenerate-random-32-chars-string
FLASK_ENVproduction

💡 Generate SECRET_KEY: python -c "import secrets; print(secrets.token_hex(32))"

4 Deploy!

Klik tombol Deploy. Vercel akan:

  1. Install dependencies dari requirements.txt
  2. Build Python runtime untuk api/index.py
  3. Route semua request ke Flask app
  4. Generate URL: esteler-app.vercel.app

⏱️ Build time: ~45 detik pertama, ~15 detik selanjutnya.

⚠️ Catatan Vercel Free Tier

  • Max execution: 10 detik per request (cocok untuk Es Teler)
  • Cold start: ~1-2 detik (pertama akses setelah idle)
  • Bandwidth: 100 GB/bulan (lebih dari cukup)
  • Custom domain: gratis (bisa connect domain sendiri)

9 Troubleshooting

❌ "FATAL: connection terminated unexpectedly"

Neon auto-suspend setelah 5 menit idle. Pastikan pakai pooled connection dan pool_pre_ping=True di config.py (sudah otomatis).

❌ "ImportError: No module named 'app'" di Vercel

Pastikan file api/index.py ada dan import app dengan path relatif yang benar. Check struktur folder di section 3.

❌ Template tidak ketemu di Vercel

Vercel filesystem read-only kecuali /tmp. Set template_folder explicit di api/index.py dengan absolute path.

❌ Build fail "psycopg2 not found"

Pakai psycopg2-binary bukan psycopg2 di requirements.txt. Yang binary udah include native libs.

❌ Login redirect loop

Cek User extend UserMixin. Cek SECRET_KEY di-set (Flask butuh untuk session cookie).

⚡ Performance tips
  • Cold start lambat? → upgrade Vercel ke Pro untuk warm instances
  • Query lambat? → tambah index pada kolom yang sering di-filter
  • Image lambat load? → host di Vercel Blob / Cloudflare R2

💡 Catatan Penting

SDD asli menyebut deployment di Railway/Render/PythonAnywhere dengan Docker dan Gunicorn. Kami memilih Vercel + Neon karena lebih cocok untuk skala UMKM (gratis, scale-to-zero, tidak perlu maintain server). Refactor ini tetap mempertahankan struktur SDD (Three-Tier, Blueprint, Service Layer), hanya beradaptasi dengan environment serverless.

Decision Tree (scikit-learn) sesuai SDD 3.2.5 belum diimplementasi karena butuh dataset training yang cukup. Saat ini pakai rule-based scoring sebagai fallback (sesuai strategi SDD: "fallback ke best seller jika dataset tidak mencukupi"). Tinggal swap function calculate_match_score() nanti.
